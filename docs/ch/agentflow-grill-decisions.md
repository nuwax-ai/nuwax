## 1. 节点类型总览

**确认 9 种节点：** 开始、结束、路由决策、智能体、插件、工作流、人工确认、询问用户、结果评估

- 插件、工作流节点与 Workflow V3 中的节点一致，仅 AgentFlow 场景下属性面板有增量改动
- ExternalConnector 在原型图中未出现，**本期不做**
- 开始/结束节点与 V3 一致，新建时自动初始化，不可删除

---

## 2. 路由决策节点（RouteDecision）

- **端口规则：** 单入，多出（动态分支）
- **默认端口：** 有一个"默认"输出端口，用于兜底
- **分支端口：** 每个 route 分支生成独立输出端口，一对一连接
- **分支管理：** 至少 1 个分支，所有分支可增删改（名称、描述均可修改）
- **独立模型选择：** 需要独立的「识别模型」选择（`nodeConfig.modelId`）
- **属性面板：** 系统提示词 + 路由分支列表（名称 + 描述）

---

## 3. 智能体节点（Agent）

- **端口规则：** 单入单出
- **无子工作流概念：** 不涉及 subFlowId 的嵌套调用
- **上下文传递：** 新增 auto/manual 模式
  - auto：自动取上游输出
  - manual：手动选择上下文来源（用户输入 / 流程上下文 / 上游节点输出 / 自定义参数）
- **补充提示词：** 可添加提示词约束输出格式，支持引用上下文变量

---

## 4. 插件节点（Plugin）

- **端口规则：** 单入单出
- **与 V3 关系：** 复用 V3 Plugin 节点，AgentFlow-only 增量：
  - 新增 auto/manual 入参模式选择
  - manual 模式下可手动绑定每个参数的值

---

## 5. 工作流节点（Workflow）

- **端口规则：** 单入单出
- **与 V3 关系：** 复用 V3 Workflow 节点，AgentFlow-only 增量：
  - 新增触发方式选择（同步等待 / 异步立即返回）
  - 新增 auto/manual 入参模式选择

---

## 6. 人工确认节点（HumanInteraction:approve）

- **端口规则：** 单入，多出（动态分支）
- **初始修正：** 原代码固定 approve/reject 两个端口，原型改为动态分支
- **分支管理：** 默认"通过"/"拒绝"两个分支，可任意添加，至少保留 1 个
- **所有分支都可以修改 name/desc**，最少 1 个分支
- **确认角色：** 使用者确认（当前用户） / 外部确认（通过通道通知他人）
- **审核模式：** 通过拒绝 / 可编辑后通过 / 仅通知自动通过
- **外部确认通道：** webhook 通道配置，支持超时处理（跳过/拒绝/重试/中止）、重试次数
- **审核说明：** 面向审核人的说明文字

---

## 7. 询问用户节点（HumanInteraction:ask）

- **端口规则：**
  - 文本回复 / 表单回复：单入单出
  - 选项回复：单入，每个选项一个独立输出端口（一对一）
- **选项端口未连接 = 放弃该选项**
- **拖拽：** 原型中 ask 和 approve 是独立的拖拽项（stencil），但底层同属 `HumanInteraction` + 不同 `hitlMode`
- **回复模式 3 种：**
  - 文本回复（text）
  - 选项回复（options）：每个选项显示为独立输出端口
  - 表单回复（form）：支持 radio/checkbox/input/number/textarea/file 等字段类型
- **3 种回复模式都需要实现**
- **上下文写入：** 用户回复自动写入全局上下文 `user_reply`，供下游节点使用

---

## 8. 结果评估节点（EvalGate）

- **端口规则：** 单入，多出（动态分支）
  - "通过"分支固定为第一个端口
  - 不通过分支可动态添加，至少 1 个
- **评估模型：** 独立模型选择（`nodeConfig.modelId`）
- **共用上下文：** 可开启"共用上游智能体上下文"（复用 `autoWirePrevOutput`）
- **评估方式：** 加权打分制（取代原有 rule/llm-judge 验证器）
  - 每个评估项目有名称 + 权重（%）+ 评估标准描述
  - 权重合计必须等于 100%
  - 通过阈值（默认 80 分）
- **评估过程输出：** 可开关，开启后将评估过程输出给用户
- **循环评估：** 最多循环评估次数（默认 3 次），不通过时重新评估
- **超限提示：** 超过上限仍未通过时，显示自定义提示语给用户
- **分支管理：** 评估分支/输出分支（人工确认）也是至少 1 个分支的逻辑

---

## 9. 连线规则汇总

| 规则          | 说明                                             |
| ------------- | ------------------------------------------------ |
| 开始/结束节点 | 与 V3 一致，新建自动创建，不可删除               |
| 默认规则      | 每个输出端口最多连接 1 个下游节点（一对一）      |
| 未连接端口    | = 流程到此结束 / 放弃该路径                      |
| 分支节点      | 路由决策、人工确认、询问用户(选项模式)、结果评估 |
| 最少分支      | 1 个                                             |
| 分支可编辑    | 所有分支都可修改 name/desc                       |
| 可选连接      | 分支端口可以不连                                 |

## 10. 全局上下文机制

- **变量语法：** `{{variableName}}`
- **解析和替换：** 后端负责
- **前端职责：** 插入或去除变量引用
- **可用变量来源：** 前端从已有节点推断
  - `startNode.nodeConfig.inputArgs` — 开始节点定义的输入变量
  - `node_{id}_output` — 各节点 outputArgs
  - `user_input` — 用户原始输入
  - `user_reply` — 询问用户节点的回复

---

## 11. 属性面板分类

- **`[REUSE]`**：复用现有 NodeConfig 字段（如 `modelId`、`autoWirePrevOutput`、`askConfig.question`）
- **`[CHANGE]`**：替代原有字段（如 `evalValidators → evalItems`、`approveNextNodeIds → branches`）
- **`[NEW]`**：需新增到 NodeConfig 的字段（如 `contextPassMode`、`confirmRole`、`replyMode`、`formFields`、`passThreshold`）
- **`[AgentFlow-only]`**：仅 AgentFlow 场景展示，V3 不受影响
